# v0.2.0 - 功能扩展与多触发类型系统

## 日期
2026-05-16

## 变更内容

### 单实例运行
- 集成 `tauri-plugin-single-instance`，再次启动时自动聚焦已有实例而非打开新窗口

### 数据管理架构
- 所有配置存放于 `%APPDATA%\com.wonderland6627.aipet\` 目录
  - `config.json` -- 全局应用设置（置顶、自启、动画速度/尺寸、当前宠物）
  - `state-config.json` -- 多触发类型的动画状态映射
  - `pets\<名称>\` -- 宠物资源（`pet.json` + `pet-atlas.json` + `spritesheet.webp`）
- 首次扫描宠物资源时自动生成 `pet-atlas.json`（帧布局），保持原 `pet.json` 不修改以兼容 Petdex/Codex 格式

### 宠物库
- Grid 列表展示所有宠物，显示第一帧缩略图与名称
- 点击宠物卡片打开详情弹窗：名称、描述、逐行动画预览（192x208）、帧数
- 所有动画预览默认自动播放
- 支持「设为当前宠物」和「刷新列表」

### 多触发类型状态配置
- 支持 6 种触发类型，每个动画状态可选择一种：
  - **当前聚焦进程** -- 预设进程列表（Unity、Rider、Cursor 等）+ 手动输入
  - **CPU/内存占用过高** -- 选择资源类型 + 百分比阈值
  - **正在播放音频** -- WASAPI 峰值检测，系统级自动识别
  - **麦克风被占用** -- WASAPI 采集端检测，自动识别会议/通话
  - **持续专注 N 分钟** -- 基于键鼠输入活跃度判定
  - **电脑挂机 N 分钟** -- 基于 `GetLastInputInfo` 空闲检测
- 触发优先级（从高到低）：麦克风 > 挂机 > 专注 > 资源占用 > 音频 > 进程聚焦 > 待机
- 唯一性校验：同一进程不可出现在多个映射中；CPU/内存/音频/麦克风/专注/挂机各只能绑定一个状态
- `running-right` / `running-left` 为拖拽专用，状态配置页自动置灰不可编辑
- 自动迁移旧版 `state-config.json`（v0.1 纯进程列表格式）

### 系统监控（Rust 后端）
- 新增 `system_monitor.rs`，每秒采集并发出 `system-state-changed` 事件
- 采集指标：前台进程名、CPU/内存使用率、音频播放状态、麦克风占用状态、空闲秒数、专注秒数
- 新增 `window_detector` 函数：`get_idle_seconds()`、`is_audio_playing()`、`is_microphone_active()`

### 应用设置
- 窗口置顶 Toggle + 开机自启 Toggle（`tauri-plugin-autostart`）
- 动画速度滑条（0.1x ~ 3x）+ 动画尺寸滑条（0.5x ~ 3x）
- 显示真实配置目录路径（如 `C:\Users\xxx\AppData\Roaming\...`）
- 「打开配置目录」按钮，直接用资源管理器打开

### 系统托盘
- 双击托盘图标打开设置窗口

### 屏幕边界限制
- 宠物浮窗拖拽时不超出屏幕边缘
- 使用所有显示器的联合区域计算边界，支持多显示器间自由拖拽

### PetCanvas 组件简化
- 移除播放/暂停交互，所有动画始终自动循环播放
