# v0.5.0 - 生图稳定性与默认策略更新

## 日期
2026-05-28

## 变更内容

本版本聚焦于 AI 宠物生成链路的稳定性和默认配置体验：一方面优化 OpenAI 生图尺寸与质量参数，减少失败率与成本；另一方面强化“基础形象确认”后的流水线控制，并修复条带拆帧时的跨帧串图问题。同时，更新了默认状态映射与桌面图标资源。

---

### 1. OpenAI 生图参数调整（更稳、更省）
- OpenAI 生成最小像素预算由 `1024x1024` 等效策略调整为 `655,360`。
- 生成尺寸现在会自动对齐到 `16` 的倍数，满足模型约束并减少异常响应。
- 接口请求默认 `quality` 从 `medium` 调整为 `low`，降低生成成本与等待时间。
- 相关实现位于 `app/src-tauri/src/ai_api_client.rs`。

### 2. 基础形象确认改为“阻塞后继续”
- 生成到基础形象后，流水线会显式等待用户确认，不再直接继续后续动作帧生成。
- 新增任务级通知机制，`confirm_base_image` 会统一唤醒等待中的生成任务。
- 当用户选择不确认时，任务会取消并安全退出。
- 相关实现位于 `app/src-tauri/src/pet_creator.rs`。

### 3. 雪碧图拆帧防串帧优化
- `extract_frames` 新增边界 gutter 裁切逻辑，减少相邻帧内容泄漏。
- 引入 `compute_gutter` 的透明度扫描策略，根据条带边界内容密度动态选择裁切宽度。
- 拆帧后仍保留内容重心回正逻辑，提升最终动作连贯度。
- 相关实现位于 `app/src-tauri/src/image_processor.rs`。

### 4. 默认状态映射与前端编辑器策略同步
- 默认触发策略整体更“轻量”：降低 CPU 阈值、缩短连续专注触发时长、调整待机时间。
- 进程聚焦默认应用集更新，更贴近日常开发与办公场景。
- 前端状态配置页默认值同步后端，避免新建映射时前后不一致。
- 相关实现位于 `app/src-tauri/src/config_manager.rs`、`app/src/windows/StateConfigSection.tsx`。

### 5. 工程与资源整理
- `pnpm-workspace.yaml` 增加 `packages: ["."]`，明确当前应用目录作为 workspace 包。
- 桌面图标资源更新（`icon.ico` / `icon.icns` / `128x128` 等），并移除一批未使用的平台图标资源。
