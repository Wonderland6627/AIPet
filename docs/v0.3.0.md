# v0.3.0 - 交互稳定性与配置体验升级

## 日期
2026-05-17

## 变更内容

### 拖拽与多显示器边界
- 宠物拖拽改为“拖拽结束后校正边界”，避免移动过程中频繁纠偏导致的卡边和抖动
- 新增命令 `clamp_main_window_to_screen`，由前端在拖拽结束时显式调用
- 新增命令 `reset_main_window_position`，支持一键重置宠物到右下角
- 边界校正算法改为“最近显示器投影纠偏”，更好支持 L/T 等不规则多屏布局
- 启动时宠物初始位置改为按主屏右下角计算，替代固定像素坐标

### 拖拽状态与动画反馈
- 拖拽期间根据窗口移动方向实时切换 `running-left` / `running-right` 动画
- 新增 `pet-dragging-changed` 事件结构：`{ dragging, direction }`
- 新增 `PetStatusPanel`，在设置页左侧实时显示当前触发状态、原因与硬件指标
- 拖拽调试信息支持显示方向（向左拖拽 / 向右拖拽）

### 应用设置页优化
- 设置页布局升级为纵向填充，底部新增版本号显示（`v{version}`）
- 新增「重置宠物位置（右下角）」按钮
- 文案微调（如“宠物置顶”“宠物位置”）
- 设置窗口尺寸调整为 `1000x700`，并设置最小尺寸 `900x650`

### 状态配置编辑器增强
- 触发条件编辑改为“原位更新”，修复编辑后条目掉到底部的问题
- 动画切换保持原顺序插回，避免列表顺序抖动
- 进程触发支持读取“正在运行窗口进程”，并显示为 `应用名（exe）`
- 进程下拉预加载优化，首次打开即有完整候选
- 触发参数区对齐优化，统一 cell 内中部区域的垂直节奏
- 保持触发类型唯一性/资源唯一性/进程唯一性校验逻辑

### 宠物库与设置导航体验
- 宠物库改为横向选择 + 右侧详情布局，支持更紧凑的动作预览网格
- 新增“使用中”状态标识与「召唤此宠物」按钮交互
- 左侧导航集成状态面板，减少状态配置调试成本

### 后端监控与进程识别
- 新增命令 `list_running_processes`，返回结构化项：`{ exe, displayName }`
- 可见窗口进程过滤：仅采集有可见顶层窗口的进程 PID
- Windows 下优先读取文件元数据 `FileDescription` 作为应用显示名（失败时回退 exe）
- `save_app_config` 增加 delta 应用逻辑，仅在 `autoStart` 变更时切换自启动状态

### 动画渲染稳定性
- `PetCanvas` 序列帧位移改为 `transform: translate3d(...)`，替代 `background-position-x`
- 增加独立条带容器、`will-change: transform` 与像素风渲染参数，减少闪烁与卡顿

## 依赖变更

### Rust
- `windows` crate 增加特性：`Win32_Storage_FileSystem`（用于读取可执行文件版本信息）

### 前端
- 无新增外部依赖

## Bug 修复
- 修复多显示器拖拽时边缘拦截，导致无法跨屏的问题
- 修复拖拽结束后窗口状态恢复不稳定导致的边界行为异常
- 修复状态配置在编辑触发条件/动画后条目顺序被重排的问题
- 修复进程下拉首次打开候选不完整、长文案显示体验不一致的问题
