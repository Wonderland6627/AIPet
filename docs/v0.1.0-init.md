# v0.1.0 - 项目初始化

## 日期
2026-05-15

## 变更内容

### 项目搭建
- 使用 Tauri 2.0 + React 18 + Vite 6 + TypeScript 5 + Tailwind CSS v4 搭建
- 项目位于 `AIPet/` 子目录，与 `templates/` 和 `docs/` 同级

### 窗口配置 (tauri.conf.json)
- **main 窗口**: 192x208px, 透明/无边框/置顶/不显示任务栏, 初始位置 (1200,600)
- **settings 窗口**: 680x480px, 标准窗口, 默认隐藏
- 点击穿透: CSS `pointer-events: none` + 宠物容器 `pointer-events: auto` (MVP 方案)

### Rust 后端 (src-tauri/src/)
- **window_detector.rs**: 使用 `windows` crate 调用 GetForegroundWindow + QueryFullProcessImageNameW 检测前台进程
- **lib.rs**: 每秒异步轮询前台进程, 变化时 emit `active-process-changed` 事件; 拦截 settings 窗口关闭改为隐藏
- **tray.rs**: 系统托盘菜单 (显示设置 / 重新孵化 / 退出)

### React 前端 (src/)
- **PetCanvas.tsx**: CSS steps() 序列帧动画, 9 行状态各自帧数动态查表
- **App.tsx**: 监听后端进程事件 + 硬编码映射 (Cursor->working, Excel->waiting 等); 拖拽用 startDragging(); 双击打开设置
- **Settings.tsx**: 左侧导航 (宠物库/职责设置/应用设置), 粉色主题高亮

### 序列帧规范 (以 pet_request.json 为准)
- 9 行 x 8 列, 单元格 192x208px, 总图 1536x1872px
- 各行帧数: idle(6) / running-right(8) / running-left(8) / waving(4) / jumping(5) / failed(8) / waiting(6) / running(6) / review(6)

## 依赖
- Rust: tauri 2.x, windows 0.61, tokio, serde
- Frontend: react 18, react-router-dom 6, @tauri-apps/api 2, tailwindcss 4, vite 6
